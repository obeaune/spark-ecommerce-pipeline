# Ecommerce Spark Pipeline

Batch data pipeline built with Apache Spark (Scala) for ingestion and transformation of e-commerce financial data, applying a bronze/silver/gold layered architecture.

## Architecture

    positional flat file (input)
            ↓
       [ Bronze ]  → raw data as Parquet
            ↓
       [ Silver ]  → cleaned and typed data
            ↓
        [ Gold ]   → aggregated financial metrics

## Layers

**Bronze** — Reads positional flat files simulating payment acquirer transactions. Parses each field by fixed position and stores raw data as Parquet without transformations.

**Silver** — Applies data typing and cleaning: converts amounts from cents to decimal, parses timestamps, maps payment type and status codes to human-readable descriptions.

**Gold** — Aggregates data by merchant, generating financial metrics: total transactions, total volume, average ticket, counts by status and volume by payment type.

## Data Generator

The input files are generated by [ecommerce-data-generator](https://github.com/obeaune/ecommerce-data-generator), a Python script that simulates positional flat files from a payment acquirer.

## How to Run

1. Generate input data using [ecommerce-data-generator](https://github.com/obeaune/ecommerce-data-generator)
2. Copy the generated `.txt` file to `data/input/`
3. Run `Main.scala` from your IDE or via Maven
